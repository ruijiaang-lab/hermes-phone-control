# X Post Polisher

## 概述

用户通过微信/Telegram 发送随笔内容，Hermes 将其润色成可直接发布的 X/Twitter 帖子。

## 触发条件

- 用户说"润色"、"发推"、"帮我写个推"
- 用户发一段文字并明确表示要发推

## 执行流程

1. 用户发送随笔内容
2. Hermes 按人设规则润色
3. 回复润色后的草稿给用户审核
4. 用户确认后保存到 X 草稿箱

## 人设规则（自定义）

> ⚠️ **你需要修改这部分**，改成你自己的账号风格和定位。

- 风格：（你的风格，如：真诚、干货、幽默）
- 定位：（你的定位，如：AI 实战分享）
- 语言：中文为主，专业词混英文
- 语气：第一人称，句子简洁有力

## 输出格式

```
【X 帖子草稿】
正文内容（线程用 1/ 2/ 3/ 编号）
---
建议配图想法：（可选）
预计互动效果：（简评）
```

## 润色规则

1. 严格保持用户原意，只润色语言、结构、吸引力
2. 不添加用户没提到的内容
3. 增加"人味"和互动感
4. 长度控制在 X 友好范围（单帖或 3-8 条短线程）
5. 专业术语保留英文，不强行翻译

## 确认词

- "ok"/"通过"/"存"/"保存" → 保存到 X 草稿箱
- "改"/"重写"/"再来" → 重新润色

## 草稿保存

确认后调用：
```bash
python3 ~/Desktop/x_save_draft.py "润色后的帖子内容"
```

## 注意

- 润色完必须给用户审核，不能自动保存/发布
- 用户确认后才执行保存
- 生图 prompt 用中文
